# Documentation, ADRs & Ubiquitous Language

- **ADRs**: capture architecture and provider decisions (context, decision, consequences).
- **Feature specs**: succinct GWT scenarios + data contracts.
- **Repo docs**: README explains features, how to run, how to test, how to deploy; link to tests to “follow the layers”.
- **Language**: maintain a living glossary per bounded context; keep names consistent with the glossary.
