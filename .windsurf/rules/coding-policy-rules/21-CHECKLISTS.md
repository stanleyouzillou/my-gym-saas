# Checklists

## Definition of Ready (DoR)

- User story with GWT scenarios
- Data contracts known
- Non‑functional requirements listed (perf, security, compliance)
- Dependencies & risks identified

## Definition of Done (DoD)

- All acceptance & unit tests passing
- Integration/E2E for critical paths
- Observability, docs, ADRs updated
- Feature flagged if risky; rollout plan defined

## Operability Readiness

- Dashboards, alerts, runbook
- Error budgets & SLOs set
- Backups & restore tested (where applicable)
